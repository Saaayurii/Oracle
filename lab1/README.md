# Лабораторная работа №1 — Создание рабочего пространства Oracle

## Цель работы

1. Получение навыков установки и запуска СУБД Oracle Database Express Edition.

---

## Выполнение (Docker-подход)

Вместо нативной установки используется официальный Docker-образ `gvenzl/oracle-xe:21-slim`,
что позволяет запустить Oracle XE 21c на любой ОС без изменения системного окружения.

### Шаг 1 — Установка Docker

```bash
# Ubuntu / Debian
sudo apt-get update
sudo apt-get install -y docker.io docker-compose-plugin
sudo usermod -aG docker $USER
```

Для Windows/macOS: скачать [Docker Desktop](https://www.docker.com/products/docker-desktop/).

### Шаг 2 — Запуск Oracle XE 21c

```bash
# Из корня репозитория
docker compose up -d

# Дождаться инициализации (3–5 минут)
docker compose logs -f oracle-xe
```

Признак готовности в логах:
```
DATABASE IS READY TO USE!
```

### Шаг 3 — Проверка подключения

```bash
docker exec -it oracle_xe sqlplus system/Oracle123@XEPDB1
```

```sql
SELECT banner FROM v$version;
```

Ожидаемый вывод:
```
Oracle Database 21c Express Edition Release 21.0.0.0.0 - Production
```

### Шаг 4 — Доступ к Oracle APEX

| Параметр  | Значение                       |
|-----------|--------------------------------|
| URL       | http://localhost:8080/apex     |
| Workspace | `INTERNAL`                     |
| User      | `ADMIN`                        |
| Password  | `Oracle123`                    |

## Контрольные вопросы

### 1. Что такое база данных?

**База данных (БД)** — организованная совокупность структурированных данных,
хранящихся в электронном виде. БД обеспечивает централизованное хранение,
управление и поиск информации по запросам пользователей и приложений.

### 2. Что такое экземпляр (сервер) базы данных?

**Экземпляр Oracle** — набор фоновых процессов и выделённых областей памяти
(SGA — System Global Area), обеспечивающих доступ к физическим файлам базы данных.
Один экземпляр работает с одной БД; в конфигурации RAC несколько экземпляров
могут совместно использовать одну БД.

### 3. Ключевые отличия в редакциях Oracle Database

| Редакция           | Ограничения / особенности                              |
|--------------------|-------------------------------------------------------|
| Express Edition (XE)| До 2 CPU, 2 ГБ RAM для БД, 12 ГБ данных, бесплатна  |
| Standard Edition 2 | До 2 сокетов, нет RAC, лицензируется по сокетам       |
| Enterprise Edition | Без ограничений, все опции (RAC, Partitioning и др.) |

### 4. Метрики лицензирования Oracle Database

- **Named User Plus (NUP)** — лицензия на конкретного пользователя (мин. 5 на процессор).
- **Processor** — лицензия на физический/виртуальный процессор (коэффициент 0.5 для Intel).

### 5. Что такое Oracle APEX?

**Oracle Application Express (APEX)** — низкокодовая веб-платформа для разработки
приложений, встроенная непосредственно в СУБД Oracle. Позволяет создавать
полноценные веб-приложения с отчётами, формами и дашбордами, используя только
браузер и SQL/PL/SQL, без внешних серверов приложений.
